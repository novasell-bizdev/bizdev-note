## 記事構成

### 冒頭パンチライン（最初の3段落以内）

> **SNS拡散の第一起爆点。** 前置きなしで本題に入る。以下の3型から1つ選択。

**型A: エピグラフ型**
```
――[1行の宣言的フレーズ]

「[リアルな現場の声・誰かの発言]」

[その声が投げかける問いの解説（2-3文）]
```

**型B: 自己論駁型**
```
[過去の常識・自分たちの前提を引用]

…で、結果はどうだったか？

[現実で冷徹に否定する（1-2文の短文断言）]
```

**型C: 挑発的問い型**
```
[業界の常識に正面からケンカを売る断言（1文）]

[その断言を裏付ける具体的な数字・エピソード（2-3文）]
```

> **品質基準**: この冒頭だけで「続きを読みたい」と思わせる。冒頭120字がメタディスクリプション代替になるため、キーワード＋要約＋ベネフィットも凝縮する。

### H2: 今回のテーマと背景

- なぜこのテーマを取り上げたのか
- チーム内での問題意識
- 発表者の視点・仮説

### H2: 議論のハイライト

#### H3: 論点1 — [具体的な論点]

- 発表者の主張
- チーム内で出た反論・追加視点
- 数値データ・事例があれば記載
- **[PUNCHLINE]**: セクション内に1つ、140文字以内の引用可能フレーズを配置

> **段落の緩急**: 長い説明段落の直後に「だが、あえて問いたい。」等の一行段落を挟む。

#### H3: 論点2 — [具体的な論点]

- 同上
- **概念再定義**: 既知の概念に新しい意味を与える箇所を1つ以上含める
  - 例: 「XではなくYだ」「Xの正体はYである」

### H2: 結論が出なかった問い

> ここが「公開日誌」の核心。完璧な答えではなく、未解決の問いを読者に投げかける。

- 「〜については、まだ答えが出ていません」
- 「皆さんの組織ではどうですか？」

### H2: まとめ

- 3行以内の結論リスト（GEO最適化）
- 次回予告（あれば）

### 締めパンチライン（最後の2-3段落）

> **SNS拡散の第二起爆点。** 分析や提言で終わらず、必ず「意志」か「覚悟」で締める。以下の2型から1つ選択。

**型A: 意志宣言型**
```
[チームとしての決意を「私たちは〜だ」の一人称宣言で表明]

[まだ道半ばであることを認めつつ、前を向く1文]
```

**型B: 覚悟+問い型**
```
[自分たちへの戒めを含む1文]

[読者も巻き込む形で問いかけ、余韻を残す]
```

> **品質基準**: 最後の1文がそのままXに引用されても成立する切れ味。

## パンチライン修辞技法リファレンス

記事全体で以下の技法を使い分ける。各セクションに最低1つ。

| 技法 | パターン | 例 |
|------|---------|-----|
| **逆説・再定義** | 「XではなくYだ」で既知の概念をひっくり返す | 「作業を仕事と呼んでいた。これからは意志を仕事と呼ぶ。」 |
| **短文断言** | 1-2文の格言的断定 | 「アウトプットに価値はない。成果だけが存在証明になる。」 |
| **メタファー一貫** | 1つの比喩を記事全体で展開し切る | マシンガン→照準→シューター→引き金 |
| **具体→抽象の跳躍** | 泥臭い数字・エピソード → 哲学的命題 | 「500件営業した」→「何かを起こすなら常に狂気的な熱狂がそこにあった」 |
| **格言の一点投入** | 先人の言葉を1箇所だけ効かせる | 「勝ちに不思議の勝ちあり、負けに不思議の負けなし。」 |
