## 1. noteのSEO特性

### 1-1. ドメインパワー（DR/DA）

noteは日本国内のプラットフォームとして**圧倒的なドメインパワー**を保有している。

| 指標 | スコア | 備考 |
|------|--------|------|
| Ahrefs DR (Domain Rating) | **91** | 国内トップクラス。上場企業レベル |
| Power Rank Check | **96.4 / 100** | 個人ブログが10-20程度なのに対し異常値 |
| Moz DA推定 | **88-90** | DRとほぼ同レベル |

**なぜこれほど高いのか:**
- 会員数777万人超（2024年時点）から生まれる膨大な記事量
- 2025年1月にGoogleと資本業務提携（約4.89億円出資）→ Gemini連携
- 広告なしのクリーンなHTML構造でページ表示速度が速い
- 実名・専門家による一次情報コンテンツの集積（E-E-A-T要素）

### 1-2. Google検索での上位表示の仕組み

noteの記事がGoogle検索で上位表示されやすい構造的な理由は以下の通り。

**プラットフォーム恩恵（サブディレクトリモデル）:**
- note記事のURLは `note.com/{ユーザー名}/n/{記事ID}` 形式
- note.comのドメイン評価をそのまま継承するため、新規の個人ブログより圧倒的に有利
- 「強いドメインの傘の下で書く」効果 → 新規記事でもインデックスが速い

**ただし注意点がある:**
- 2025年のGoogleコアアップデートで**二次情報・まとめ記事の評価が下落**
- noteであっても、独自性のない記事はドメインパワーだけでは上位表示が難しくなっている
- 一次情報（実体験、独自調査、専門知見）を含む記事ほど評価が高い
- YMYL分野では専門性・権威性がより重視されるようになっている

### 1-3. note内検索 vs Google検索の最適化の違い

| 要素 | note内検索 | Google検索 |
|------|-----------|------------|
| 主な入口 | ハッシュタグ、トピック、フォロー | キーワード検索 |
| 重要な指標 | スキ数、フォロワー数、更新頻度 | E-E-A-T、被リンク、コンテンツ品質 |
| タグの効果 | **非常に大きい** — カテゴリ分類の主軸 | 間接的（note内リンク構造として認識される程度） |
| 最適化の焦点 | 読者エンゲージメント、SNS拡散 | 検索意図への合致、見出し構造 |
| 流入比率 | note全体の約40% | note利用者の62.4%がGoogle検索経由の流入増を報告 |

**両方を狙う戦略:**
- タイトルにキーワードを含める（Google最適化）
- 人気タグを5つ設定（note内最適化）
- 冒頭で検索意図に直接回答（Google最適化）
- 記事末尾で関連記事を紹介（両方に効果的）

### 1-4. 技術仕様

| 項目 | noteの仕様 | ユーザーが制御可能か |
|------|-----------|------------------|
| **canonical URL** | 自動設定。各記事に正規URLが付与される | 不可（プラットフォーム管理） |
| **noindex** | 公開記事は自動的にindex。個別noindex不可 | 不可 |
| **meta description** | 記事冒頭から自動生成。別途設定不可 | 不可（冒頭を工夫するしかない） |
| **title tag** | 記事タイトルがそのまま使用される | タイトル入力で間接制御 |
| **OGP (Open Graph)** | 自動設定。見出し画像がOGP画像になる | 見出し画像の選択で間接制御 |
| **OGP画像サイズ** | 推奨: 1200x630px（1.91:1） | 可（画像アップロード時に調整） |
| **見出しタグ** | 「見出し」= H2、「小見出し」= H3 | 可 |
| **画像alt属性** | 画像メニューから「代替テキスト」設定可 | **可** |

| **robots.txt** | プラットフォーム管理。ユーザー設定不可 | 不可 |
| **外部リンク属性** | 自動でnofollow付与（一部dofollow説あり） | 不可 |
| **ページ速度** | 広告なし + 軽量HTMLで高速 | N/A |

## 2. note記事のSEO最適化テクニック

### 2-1. タイトルの最適化

**推奨文字数: 30〜35文字**

- 35文字を超えると検索結果でタイトルが途切れる
- **対策キーワードはタイトルの前半（左側）に配置**
- 数字・具体性を含めるとCTR向上（例:「3つの秘訣」「5つのステップ」）
- 疑問形やハウツー形式も効果的

**良い例:**
- 「AI時代のBizDev生存戦略｜工数1/3で売上3倍の実践法」（31文字）
- 「Claude Codeで業務効率化｜非エンジニアでもできる5つの活用法」（31文字）

**悪い例:**
- 「AIについて思うこと」（キーワードが曖昧、検索意図不明）
- 「【保存版】【2026年最新】完全解説！AI×事業開発の...」（修飾過多で途切れる）

### 2-2. メタディスクリプション（概要文）

**noteの仕様: 個別設定不可。記事冒頭から自動生成。**

以前は公開設定画面の「詳細設定」から「検索結果用の説明文」を設定できるとの情報もあったが、現在のnoteでは**冒頭テキストからの自動生成が基本**。

**対策:**
- **記事の冒頭1〜2文**に以下を盛り込む:
  - 対策キーワード
  - 記事の要約（何が得られるか）
  - 読者へのベネフィット
- 目安: 冒頭120文字以内に上記要素を凝縮

**冒頭テンプレート:**
```
[テーマ]について、[具体的な成果/数値]を達成した実践経験をもとに解説します。
[ターゲット読者]が[得られるベネフィット]を体系的にまとめました。
```

### 2-3. 見出し（H2/H3）へのキーワード配置

**noteでの設定方法:**
- テキスト選択 → 「見出し」= **H2タグ**
- テキスト選択 → 「小見出し」= **H3タグ**
- H1は記事タイトルが自動設定

**キーワード配置ルール:**
1. H2には章のメインテーマ + 対策キーワードを含める
2. H3でH2を細分化。関連キーワード・ロングテールを配置
3. 階層構造（H2→H3）を崩さない（H3の直後にH2を飛ばさない）
4. 見出しだけで記事の全体像が把握できる構成が理想

**見出し構成例（AI×BizDev記事）:**
```
H1: AI時代のBizDev生存戦略（タイトル・自動）

H2: なぜBizDevにAI活用が不可欠なのか
  H3: 事業開発の工数が3倍に膨らんだ背景
  H3: AI導入で変わる意思決定プロセス

H2: 非エンジニアでもできるAI活用の5つの実践法
  H3: Claude Codeで市場調査を自動化
  H3: Cursorで簡易プロトタイプを構築
  H3: ChatGPTで競合分析レポートを生成

H2: AI×BizDevの成功事例3選
  H3: 工数1/3で売上3倍を達成した方法
  H3: SaaS企業が実践するAIネイティブ営業

H2: まとめ：AI時代に生き残るBizDevの条件
```

### 2-4. 画像のalt属性

**noteでの対応状況: 設定可能**

画像アップロード後、画像メニューから「代替テキスト」を入力できる。

**alt属性の書き方:**
- 画像の内容を50〜80文字で簡潔に説明
- 対策キーワードを自然に含める（ただし詰め込みはペナルティリスク）
- 装飾画像には空のaltで可

**例:**
- 良い: `alt="Claude Codeの実行画面。ターミナルからAIエージェントがコードを自動生成している様子"`
- 悪い: `alt="画像"` / `alt="AI AI AI BizDev 事業開発"`

### 2-5. 内部リンク・外部リンクの効果

**内部リンク（note記事間リンク）:**
- 自分の過去記事への埋め込みリンクで回遊率向上 → 滞在時間増加 → SEO評価向上
- noteの埋め込み機能（URLペーストでカード表示）が便利
- 関連テーマのシリーズ記事を相互リンクすることで、トピッククラスターを形成

**外部リンク:**
- noteから外部サイトへのリンクには**nofollow属性が自動付与**される
  - ただし2025年以降、Googleはnofollowを「ヒント」として扱うため、完全に無効ではない
- 信頼性の高い外部ソース（公式サイト、論文、統計）へのリンクは、記事のE-E-A-T向上に寄与
- 自社サイト → note、note → 自社サイトの相互リンクは、ドメイン分散戦略として有効

**被リンク戦略:**
- note記事が高品質であれば、SNS共有経由で**自然な被リンク**を獲得できる
- noteのドメインパワー（DR91）の恩恵を受けられるのは最大のメリット
- 逆に自社ブログからnoteへの被リンクは、noteのドメインパワーが高すぎるため効果は限定的

### 2-6. タグ（ハッシュタグ）の選び方とSEO効果

**設定方法:** 記事公開時に最大5つまで設定

**SEO効果:**
- **note内検索には大きな効果** → 同タグの記事一覧に表示される
- **Google検索への直接効果は限定的** → タグは内部リンクの一種として認識される程度
- 同タグ記事群としてGoogleに認識されることで、間接的にトピック専門性をアピール

**タグ選定のポイント:**
1. note内で検索されやすい人気タグを選ぶ（例: `#AI活用`, `#事業開発`, `#BizDev`）
2. ニッチすぎるタグや独自造語タグは避ける
3. 記事内容と一致するタグに絞る（関連性の低いタグはマイナス）
4. Googleトレンドやnote内検索で需要を確認

**推奨タグ例（AI×BizDev記事の場合）:**
- `#AI活用` `#事業開発` `#BizDev` `#生成AI` `#業務効率化`

## 3. noteのGEO（Generative Engine Optimization）対応

### 3-1. AI検索でのnote記事の引用状況

**COOD株式会社の2025年通年AI引用信頼性調査** により、noteはAI検索において非常に強いポジションを確立していることが判明した。

| 順位 | ドメイン | AI引用率 |
|------|---------|----------|
| 1位 | Wikipedia | **28.4%** |
| **2位** | **note.com** | **19.2%** |
| 3位以降 | Yahoo.co.jp, YouTube等 | 各10%未満 |

Ahrefsの調査でも**3位**にランクインしており、複数調査で一貫して高評価。

**noteがAI引用元として強い5つの理由:**

1. **HTML構造の標準化** — 全記事が統一されたシンプルなHTML構造。AIクローラーが「どの記事でも同じ構造から情報を抽出できる」ため解析しやすい
2. **E-E-A-T要素** — 実名・顔出し専門家の一次情報が集積。AIの信頼性評価に直結
3. **Google提携** — 2025年のGoogle資本業務提携＋Gemini統合で技術的親和性が高い
4. **高ドメインパワー** — DR92-97でAIクローラーの巡回頻度が高く、新情報認識が迅速
5. **一次情報の宝庫** — 「自分のことを書く」文化で、実体験ベースの独自知見が蓄積

### 3-2. note記事をAI検索に引用されやすくする構造

GEO研究（Princeton/Georgia Tech/AI2）の知見とnoteの特性を組み合わせた最適化戦略:

**構造面:**
- **結論ファースト** — 冒頭3行以内に簡潔なサマリーを置く。AIは前置きが長い記事を引用しにくい
- **見出し階層の整合性** — H2→H3の論理的な構造。AIはタイトル・見出しの階層から記事全体の意図を読み取る
- **リスト・テーブルの活用** — 情報を構造化し、AIが重要ポイントを識別しやすくする。YMYL関連クエリでは引用記事の91%がリストを含んでいた
- **FAQ形式** — 「Q: ...」「A: ...」の形式はAIが抽出しやすい

**コンテンツ面:**
- **統計データ・数値の引用** — 統計データの追加でAI引用率が**40%以上向上**（GEO研究）
- **一次情報** — 自分の実体験、独自調査、オリジナルデータを含める
- **専門用語の定義** — 初出の専門用語は括弧書きで定義。AIの理解精度向上
- **出典明記** — 信頼できる外部ソースへの参照を含める

**最適なコンテンツ構造テンプレート:**
```
【概要】3行以内の簡潔なサマリー
【詳細解説】専門用語を含む詳細説明
【具体例】実際の事例やデータ（数値入り）
【参考文献】信頼できる外部ソースへのリンク
```

### 3-3. 構造化データ/Schema Markupのnoteでの対応状況

| 項目 | 状況 |
|------|------|
| Article Schema | プラットフォーム側で自動実装 |
| Person Schema | 著者プロフィールから自動生成 |
| BreadcrumbList | 自動実装 |
| FAQ Schema | **非対応**（ユーザーが追加不可） |
| HowTo Schema | **非対応** |
| カスタムJSON-LD | **追加不可**（HTMLの直接編集不可） |

**制約と対策:**
- noteではSchema Markupのカスタマイズができない
- これはデメリットだが、プラットフォーム側の基本実装がしっかりしているため、大きな問題にはならない
- FAQ形式のコンテンツを書くことで、AIがFAQ的に情報を抽出する効果は期待できる（Schema無しでも）
- **自社サイトとの併用**が推奨: 自社サイトにはカスタムSchema Markupを実装し、noteは一次情報の発信基地として分業

### 3-4. AI検索時代のnote活用戦略（ドメイン分散）

COOD調査から見えたベストプラクティス:

| 拠点 | 役割 | 強み |
|------|------|------|
| 自社サイト | 公式情報、サービス説明、構造化データ | Schema Markup自由、ブランドコントロール |
| note | 一次情報、実践知見、体験談 | AI引用率2位、DR91のドメインパワー |
| PR TIMES等 | プレスリリース、ニュース性の高い情報 | AI引用元TOP3常連 |

**相互リンクによる信頼シグナルの増幅:**
自社サイトとnoteを相互にリンクすることで、AIに対して「一貫性のある、多面的な情報源」として認識されやすくなる。

## 5. 戦略的提言

### 5-1. 「AI × BizDev」note戦略のポジショニング

BizDev朝会プロジェクトのnote展開を念頭に、以下のポジショニングを推奨:

**キーコンセプト: 「AIネイティブBizDev集団の公開実験日誌」**

他社が「AIツールの紹介」「AIの可能性を語る」レベルに留まっている中、**実際に使い倒した一次情報（数値・失敗談含む）**を出すことで差別化。

### 5-2. SEO × GEO 両取り記事構成テンプレート

```
タイトル: [キーワード] + [具体的成果/数値] + [ターゲット]（30-35文字）

冒頭120文字: 結論ファーストで要約（メタディスクリプション対策）

H2: 背景・課題（Why）
  H3: 具体的な数値で課題を提示
  H3: なぜ今この問題が重要なのか

H2: 実践した方法（How）— メインキーワード含む
  H3: ステップ1（具体名・ツール名含む）
```
