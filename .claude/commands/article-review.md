# 記事レビュー（品質チェック・機密チェック）

note記事ドラフトの品質・機密チェックを実行する。6軸評価 + 機密3段階判定 + SNS拡散力チェック。

エージェント定義: `.claude/agents/article-review.md`

## 引数

$ARGUMENTS — 以下のいずれか:
- 記事ドラフトのファイルパス（例: `articles/drafts/01_marketing-ax.md`）
- 省略時: ユーザーにパスまたはテキスト貼り付けを依頼

## 実行手順

1. `.claude/agents/article-review.md` を読み込み、チェック仕様を把握する

2. `$ARGUMENTS` を解析:
   - ファイルパスの場合 → Read ツールで記事を読み込む
   - テキスト貼り付けの場合 → そのまま使用
   - 省略時 → ユーザーに入力を依頼

3. エージェント定義に従い、以下の順序でチェックを実行:
   - **Step 2: 機密チェック（軸4）を最優先で実行**
   - 機密FAIL → 即停止、該当箇所をハイライトしてユーザーに報告
   - 機密PASS → Step 1: 構成/読みやすさ/議論可視化/ブランド/連載チェック実行
   - Step 1.5: SNS拡散力チェック（パンチライン品質）実行

4. レポートをユーザーに提示:
   - 各軸の評価（5段階）+ 機密PASS/FAIL + SNS拡散力PASS/FAIL
   - FAIL項目には修正提案を併記
   - 機密3段階判定（🟢/🟡/🔴）の詳細
   - SNS拡散力FAILの場合はパンチライン強化の具体案

5. 記事と同ディレクトリに `quality_report.md` を保存（ファイルパス指定時）

## `/note-article` からの呼び出し

`/note-article` の Phase 4 で Agent ツール経由で呼び出される場合:
- PASS → Phase 4完了を報告
- FAIL → 修正指示を Article Writer に返却（最大2回差し戻し）
- 機密FAIL → 即停止・ユーザーにエスカレーション
- SNS拡散力FAIL → パンチライン強化指示付きで差し戻し
- 3回目FAIL → ユーザーにエスカレーション
